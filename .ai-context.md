### Serviços (27 total)

Todos estão em `src/app/services/` com nomenclatura em português BR:

**Métodos Padrão CRUD:**
- `listar(page, size, sort)` - Lista com paginação
- `listarTodas()` - Lista completa (sem paginação)
- `obter(id)` - Obtém um item
- `salvar(item)` - Cria novo ou atualiza (POST/PUT)
- `atualizar(item)` - Explicitamente atualizar (PUT)
- `excluir(id)` - Deleta um item

**Exemplo:**
```typescript
// ✅ Correto
export class ContasFixasService {
  listar(page: number, size: number): Observable<Page<ContaFixa>> { }
  obter(id: number): Observable<ContaFixa> { }
  salvar(item: ContaFixa): Observable<ContaFixa> { }
}

// ❌ Evitar
getContas(), createConta(), deleteConta()  // inglês
```

### Standalone vs Módulos

**Status Atual (v2.1.0):**
- ✅ Maioria é `standalone: true` (Angular 19 style)
- ⏳ Em refatoração: `categoria.module.ts` e `cartao.module.ts` (lazy-loaded)

**Novo componente?**
```typescript
@Component({
  selector: 'app-meu-componente',
  standalone: true,
  imports: [CommonModule, ReactiveFormsModule, ...SALVE_COMMON, ...SALVE_FORMS],
  templateUrl: './meu-componente.component.html'
})
export class MeuComponenteComponent { }
```

### PrimeNG Obrigatórios

**Design System**: PrimeNG 19 (Aura/teal)
**CSS**: Zero CSS local em `.component.css` (vazio)
**Layout**: PrimeFlex (grid, flex classes)
**Ícones**: PrimeIcons (`pi-nome-icone`)

```html
<!-- ✅ Correto -->
<p-button label="Salvar" icon="pi pi-save" styleClass="w-full"></p-button>

<!-- ❌ Evitar -->
<button class="w-full">Salvar</button>  <!-- Material style -->
<p-button label="X" class="w-full"></p-button>  <!-- class não funciona em p-button -->
```

## 📦 Imports

**Padrão**: Use path aliases, NUNCA imports relativos

```typescript
// ✅ Correto
import { ContaService } from '@services/conta.service';
import { ContaFixa } from '@models/conta.model';
import { PageHeaderComponent } from '@components/shared';

// ❌ Evitar
import { ContaService } from '../../services/conta.service';
import { ContaFixa } from '../models/conta.model';
import { PageHeaderComponent } from '../shared/page-header.component';
```

**Path Aliases Disponíveis:**
- `@services/*` → `src/app/services/*`
- `@components/*` → `src/app/components/*`
- `@models/*` → `src/app/models/*`
- `@guards/*` → `src/app/guards/*`
- `@directives/*` → `src/app/directives/*`
- `@shared/*` → `src/app/shared/*`
- `@utils/*` → `src/app/utils/*`
- `@environments/*` → `src/environments/*`

## 🔧 Convenções de Código

### Idioma

- **Domínio**: Português BR (ex: `conta`, `fatura`, `provento`)
- **Técnico**: Inglês (ex: `component`, `service`, `guard`)
- **Métodos**: Português BR (ex: `listar()`, `salvar()`)
- **Variáveis**: Português BR (ex: `saldoTotal`, `dataVencimento`)

### Comentários

- ✅ **Sim**: Por quê (constraints não óbvias, workarounds)
- ❌ **Não**: O quê (código bem nomeado se explica)

```typescript
// ✅ Bom
// Aguarda 200ms para o DOM atualizar antes de trigger scroll
setTimeout(() => { }, 200);

// ❌ Evitar
// Define timeout
setTimeout(() => { }, 200);
```

### Estado Reativo (RxJS)

**Padrão obrigatório:**
```typescript
export class ContasFixasService {
  private financasChanged$ = new BehaviorSubject<void>(undefined);

  salvar(item: ContaFixa): Observable<ContaFixa> {
    return this.http.post(...).pipe(
      tap(() => this.financasChanged$.next(undefined))
    );
  }

  obterFinancasChanged(): Observable<void> {
    return this.financasChanged$.asObservable();
  }
}
```

**Componente:**
```typescript
export class ListContasFixasComponent implements OnInit {
  contas$ = this.contasFixasService.obterFinancasChanged().pipe(
    switchMap(() => this.contasFixasService.listar(0, 100, ''))
  );

  constructor(private contasFixasService: ContasFixasService) {}
}
```

## 🚀 Fluxo de Trabalho para IAs

### 1. Antes de Começar

Sempre ler:
- [ ] `CLAUDE.md` (padrões obrigatórios do projeto)
- [ ] `CONVENTIONS.md` (12 seções de nomenclatura)
- [ ] Este arquivo (`.ai-context.md`)

### 2. Tipos de Tarefas Comuns

#### A. Novo Componente

```bash
# 1. Criar pasta em domínio correto
src/app/components/[DOMINIO]/[meu-componente]/

# 2. Arquivos necessários
- meu-componente.component.ts (standalone: true, imports: [...])
- meu-componente.component.html (template)
- meu-componente.component.spec.ts (testes)

# 3. Path alias no import
import { MeuComponenteComponent } from '@components/[DOMINIO]/meu-componente/meu-componente.component';

# 4. Build + teste
npx ng build --configuration development
```

#### B. Novo Serviço

```bash
# 1. Arquivo em src/app/services/
meu-servico.service.ts

# 2. Estrutura
@Injectable({ providedIn: 'root' })
export class MeuServicoService {
  listar(page, size, sort): Observable<Page<Item>> { }
  obter(id): Observable<Item> { }
  salvar(item): Observable<Item> { }
  excluir(id): Observable<void> { }
}

# 3. Atualizar imports de componentes
import { MeuServicoService } from '@services/meu-servico.service';
```

#### C. Refatoração

**Sempre em commits organizados:**
```bash
git commit -m "refactor(services): padronizar métodos em PT-BR"
git commit -m "refactor(components): padronizar nomes com prefixo list-"
git commit -m "refactor(imports): converter relativos para aliases"
```

### 3. Antes de Commitar

**Checklist:**
- [ ] `npx ng build --configuration development` passa
- [ ] 0 erros, 0 warnings
- [ ] Imports usam aliases (`@services/`, `@components/`, etc.)
- [ ] Nomes seguem convenção (PT-BR para domínio, EN para técnico)
- [ ] Componentes standalone quando novo
- [ ] PrimeNG usado para UI, não Material
- [ ] CSS vazio em `.component.css`
- [ ] Commit message descritivo e bem categorizado

### 4. Padrão de Commit

```
<tipo>(<escopo>): <descrição breve>

<corpo detalhado se necessário>

```

**Tipos:**
- `feat`: Nova funcionalidade
- `fix`: Correção de bug
- `refactor`: Reorganização/padrões
- `chore`: Build, versionamento, docs
- `test`: Testes

**Exemplo:**
```
refactor(services): padronizar métodos em português BR

Convertendo métodos em inglês para português BR em 8 serviços.
Atualiza 30+ componentes consumidores com novos nomes.

```

## 🛑 Armadilhas Conhecidas (NÃO repita)

### 1. PrimeNG `class` vs `styleClass`

```html
<!-- ❌ ERRADO - não funciona em componentes -->
<p-button class="w-full">Salvar</p-button>
<p-select class="w-full"></p-select>

<!-- ✅ CORRETO -->
<p-button styleClass="w-full">Salvar</p-button>
<p-select styleClass="w-full"></p-select>

<!-- Para p-password, usar também inputStyleClass -->
<p-password styleClass="w-full" inputStyleClass="w-full"></p-password>
```

### 2. Grid + Gap

```html
<!-- ❌ ERRADO - gap quebra colunas -->
<div class="grid gap-3">
  <div class="col-6">Conteúdo</div>
</div>

<!-- ✅ CORRETO - sem gap -->
<div class="grid">
  <div class="col-6">Conteúdo</div>
</div>
```

### 3. Imports de Módulos em Standalone

```typescript
// ❌ ERRADO - módulo em standalone
@Component({
  standalone: true,
  imports: [CategoriaModule]  // ❌ Módulos não vão aqui
})

// ✅ CORRETO - componentes e directives
@Component({
  standalone: true,
  imports: [CommonModule, FormsModule, CategoriaContainerComponent]
})
```

### 4. Templates com `*ngIf` em Lazy Tables

```html
<!-- ❌ ERRADO - causa loop infinito -->
<div *ngIf="loading"><p-progressspinner></div>
<p-table *ngIf="!loading" [lazy]="true">

<!-- ✅ CORRETO -->
<p-table [lazy]="true" [loading]="loading">
```

## 🔐 Segurança & Performance

- ✅ **HttpClient sempre via serviço** (nunca direto em componente)
- ✅ **ErrorHandlerService** para tratamento de erros
- ✅ **unsubscribe** em `ngOnDestroy` ou use `| async` pipe
- ✅ **Type safety**: Nunca `any`, sempre interfaces/enums tipadas
- ❌ **Evitar**: `ngAfterViewInit` sem necessidade, memory leaks
